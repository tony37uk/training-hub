Setting Up GitHub Pages

⏱️ 15–20 minutes 🟢 Beginner friendly 💸 100% free

GitHub Pages lets you host a website completely free, directly from a GitHub repository. No server, no hosting fees, no complicated setup. This guide walks you through everything from creating an account to having your site live on the internet.

💡 What you'll end up with: A live public URL like https://your-username.github.io/your-site that anyone can visit.

Before you start

Make sure you have the following ready:

  • ✓ A computer with a web browser
  • ✓ An email address (to create a GitHub account)
  • ✓ Your website files (e.g. an index.html file)
  • ✓ About 15–20 minutes

Step 1 — Create a free GitHub account

1

Go to github.com

Open your browser and navigate to github.com. Click the Sign up button in the top right corner.

2

Fill in your details

Enter your email address, choose a password, and pick a username. Your username will appear in your site's URL, so choose something you're happy with — e.g. tonysmith.

💡 Keep your username lowercase with no spaces. Hyphens are fine: tony-smith works well.
3

Verify your email

GitHub will send you a verification email. Click the link inside it to activate your account, then sign in.

Step 2 — Create a new repository

A repository (or "repo") is where GitHub stores your website files. Think of it like a folder in the cloud.

1

Click "New repository"

Once logged in, click the + icon in the top right corner, then select New repository.

2

Name your repository

Give your repo a name — this will be part of your URL. For example, naming it training-hub will give you the URL:

https://your-username.github.io/training-hub
⚠️ Make sure to set the repo to Public. GitHub Pages only works with public repositories on free accounts.
3

Initialise the repository

Tick the box that says Add a README file. This creates the repo with a starting file, which is needed before you can upload anything.

Then click Create repository.

Step 3 — Upload your website files

1

Open the upload screen

Inside your new repository, click Add file → Upload files.

2

Drag and drop your files

Drag your index.html (and any other files like images or CSS) into the upload area. GitHub will show them queued for upload.

💡 Your main page must be named index.html. GitHub Pages looks for this file automatically when someone visits your URL.
3

Commit the files

Scroll down to the Commit changes section. You can leave the default message or type something like Add website files. Click Commit changes.

Step 4 — Enable GitHub Pages

1

Go to Settings

In your repository, click the Settings tab (the gear icon near the top of the page).

2

Find the Pages section

In the left-hand sidebar, scroll down and click Pages.

3

Set the source branch

Under Source, click the dropdown and select Deploy from a branch. Then set the branch to main and the folder to / (root).

Click Save.

4

Wait a moment

GitHub takes 1–3 minutes to build and publish your site. Refresh the Settings → Pages page after a minute and you'll see a green banner with your live URL.

🎉 Your site is live! The URL will look like: https://your-username.github.io/your-repo-name

Step 5 — Updating your site in future

Whenever you want to make changes to your site, just upload the updated files the same way — go to your repo, click Add file → Upload files, and commit again. GitHub Pages will automatically redeploy within a couple of minutes.

💡 Pro tip: If you install GitHub Desktop (free app), you can drag files from your computer and sync changes without using the website every time. Great next step once you're comfortable with the basics!

Troubleshooting

❓

My URL shows a 404 error

Check that your main file is named exactly index.html (all lowercase). Also make sure Pages is pointing to the correct branch in Settings.

❓

My changes aren't showing up

GitHub Pages can take a few minutes to update. Try a hard refresh in your browser (Ctrl + Shift + R on Windows, Cmd + Shift + R on Mac) to clear the cache.

❓

Images or styles aren't loading

Make sure all file paths in your HTML are relative (e.g. images/photo.jpg) not absolute. Also check that all filenames match exactly — GitHub is case-sensitive.
