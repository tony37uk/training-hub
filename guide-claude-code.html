📖 How-To Guide

Getting Started with Claude Code

⏱️ 20–30 minutes 🟡 Some terminal knowledge helpful 🤖 Built by Anthropic

Claude Code is Anthropic's AI coding assistant that runs directly inside your terminal. Unlike browser-based AI tools, it connects to your actual project files and can read your code, write new code, fix bugs, and even handle Git workflows — all through plain English conversation. Think of it as having a highly capable developer sitting alongside you as you work.

🎯

What you'll achieve: Install Claude Code, authenticate your account, run your first session, and learn the key commands to get productive quickly.

✨ What Claude Code can do

Before you install, here's a taste of what Claude Code can help you with once it's running:

🏗️

Build features

Describe what you want in plain English and it writes the code across multiple files.

🐛

Debug problems

Paste an error message or describe what's broken — it finds and fixes the issue.

🗺️

Understand code

Ask "what does this file do?" or "explain the database schema" and get clear answers.

🔀

Handle Git

Create branches, write commit messages, and manage pull requests through conversation.

📝

Write tests

Ask it to write unit tests for your code or check coverage gaps.

♻️

Refactor safely

Restructure and improve existing code while preserving how it works.

🛡️

Safety first: Claude Code always asks for your permission before modifying any files. You can approve changes one at a time, or enable "Accept all" mode for a session when you're confident. Nothing changes on your machine without your say-so.

📋 Before you start

Make sure you have the following in place before installing:

  • ✓ A Claude.ai account — Pro or Max subscription recommended (gives you the best usage limits). You can also use an Anthropic Console API account with pre-paid credits.
  • ✓ A supported operating system: macOS 10.15+, Ubuntu 20.04+ / Debian 10+, or Windows 10+ (with Git for Windows or WSL)
  • ✓ A terminal — Terminal on Mac, PowerShell or CMD on Windows, or any Linux terminal
  • ✓ An internet connection (Claude Code calls the Anthropic API to work)
  • ✓ Windows users only: Install Git for Windows before proceeding
⚠️

Heads up on cost: If you use a Claude.ai Pro or Max subscription, Claude Code usage is included. If you use an API account instead, usage is billed by token consumption and can add up quickly in long sessions. Start with a subscription if you're unsure.

💾 Step 1 — Install Claude Code

Claude Code uses a native installer — one command and you're done. Choose the tab for your operating system:

🍎

macOS or Linux

Open Terminal and run this single command:

# Install Claude Code (stable version)
$ curl -fsSL https://claude.ai/install.sh | bash

The installer will download and set everything up automatically. It should take under a minute.

🪟

Windows (PowerShell)

Open PowerShell and run:

# Install Claude Code on Windows
PS> irm https://claude.ai/install.ps1 | iex

Make sure Git for Windows is already installed before running this. Claude Code uses Git Bash internally on Windows.

💡

Claude Code auto-updates in the background when installed this way, so you'll always have the latest version without doing anything extra.

🔐 Step 2 — Log in to your account

1

Open your project folder in the terminal

Navigate to a folder you'd like to work in. If you're just exploring, create a new test folder:

# Create and enter a test folder
$ mkdir my-test-project
$ cd my-test-project
2

Launch Claude Code

Type claude and press Enter:

$ claude

The first time you run this, Claude Code will open a browser window and prompt you to log in with your Anthropic account. Follow the on-screen instructions to complete authentication.

3

Choose your login method

You'll be asked to choose between:

  • →Claude.ai subscription — best for most people. Uses your existing Pro or Max plan.
  • →Anthropic Console — for API access with pre-paid credits. Gives you a dedicated "Claude Code" workspace for cost tracking.

Once logged in, your credentials are saved and you won't need to log in again.

🎉

You're in! You'll see the Claude Code welcome screen with your session info and recent conversations. You're now ready to start your first session.

🚀 Step 3 — Your first session

Claude Code is conversational — just type what you want in plain English. Here are some great things to try in your first session:

💬

Ask Claude to understand your project

Before asking Claude to make changes, it's good practice to let it read and summarise your project first. This gives it the context it needs to work accurately.

claude> What does this project do? Give me a summary of the main files.
claude> Explain the folder structure of this project.
🏗️

Ask it to build something

Describe what you want clearly — the more context you give, the better the output. Claude will show you its plan and ask for permission before writing anything.

claude> Create a simple HTML page with a contact form.
           Include fields for name, email, and message.
           Add basic CSS styling to make it look clean.
🐛

Ask it to debug something

Paste an error message or describe a problem and Claude will investigate and suggest a fix:

claude> I'm getting this error when I run the app:
           TypeError: Cannot read properties of undefined (reading 'map')
           It happens in the ProductList component. Can you find and fix it?
💡

Be specific: Instead of "fix the bug", try "fix the login bug where users see a blank screen after entering the wrong password." The more detail you give, the more accurate Claude's response will be.

⌨️ Step 4 — Key commands to know

Inside a Claude Code session, you can use slash commands to control the session. Here are the most important ones for beginners:

Command What it does
/helpShows all available commands
/clearClears the conversation history — use this when starting a new task to keep Claude focused
/planSwitches to Plan Mode — Claude thinks through the problem and shows you a plan before doing anything
/reviewAsks Claude to review your recent changes for bugs, performance issues, and best practices
/costShows your token usage and estimated cost for the current session
/compactCompresses a long conversation to free up context space
/modelSwitches between Claude models — Sonnet 4.6 for most tasks, Opus 4.6 for complex work
/loginSwitch accounts or log in again if needed
Shift + TabEnter Plan Mode (same as /plan) — Claude reads files but won't edit anything until you approve
?Shows keyboard shortcuts and quick help inside a session

📄 Step 5 — Set up your CLAUDE.md file

One of the most powerful things you can do early on is create a CLAUDE.md file in your project's root folder. Claude reads this automatically at the start of every session, so it immediately understands your project without you needing to explain it each time.

1

Auto-generate a starting CLAUDE.md

Run the /init command in your session. Claude will scan your project and create a CLAUDE.md file automatically:

claude> /init
2

Customise it for your needs

Open the generated file and add your own notes — coding conventions, preferred libraries, things Claude should always or never do. Here's an example:

CLAUDE.md 
# My Project

## What this project does
A training hub website built with plain HTML, CSS and JS.
Hosted on GitHub Pages.

## Conventions
- Always use Nunito and Poppins fonts
- Keep all styles in a single <style> block
- Prefer CSS variables for colours
- Never use frameworks like React or Vue
💡

Commit your CLAUDE.md to your Git repository so your whole team benefits from the shared context — everyone gets the same consistent Claude behaviour on that project.

🧠 Tips for getting the best results

  • 1Start each new task with /clear. Context pollution (leftover context from earlier in a session) is the most common reason Claude starts giving worse answers. Clearing regularly keeps things sharp.
  • 2Use Plan Mode for big changes. Before asking Claude to refactor something large or touch multiple files, press Shift+Tab to enter Plan Mode. Review Claude's plan before it makes a single edit.
  • 3Commit before risky changes. Claude won't push to Git without asking, but having a clean Git state before a big task means you can always roll back easily.
  • 4Use Sonnet 4.6 for most tasks, and switch to Opus 4.6 (via /model) only for the hardest problems — large-scale refactoring, complex architecture decisions, or very large codebases. Sonnet is faster and cheaper for everyday work.
  • 5Ask follow-up questions. If the first response isn't quite right, don't start over — tell Claude what to adjust. "Make it shorter", "Add error handling", or "Use a different approach" all work well.

⚠️ Troubleshooting

❓

The claude command isn't found after install

Close and reopen your terminal window after installation — this refreshes your PATH so the system can find the new command. On Mac you may also need to run source ~/.zshrc or source ~/.bashrc.

❓

Windows: Git Bash not found

Claude Code needs Git for Windows to run on Windows. Download and install it from git-scm.com, then restart your terminal. If it still can't find Git Bash, you can set the path manually in your settings.json file — see the official docs for details.

❓

Claude isn't following my CLAUDE.md instructions

The most common cause is that Claude cached the file before you edited it. Run /clear to reset the conversation, or start a fresh session. Claude will re-read the updated file at the start of the new session.

❓

I've hit my usage limit

Claude.ai subscription plans have usage windows — typically a 5-hour rolling window and a weekly cap. If you hit the limit, you'll need to wait for the window to reset. Using /clear between unrelated tasks and /compact for long sessions helps you stay within limits longer.
